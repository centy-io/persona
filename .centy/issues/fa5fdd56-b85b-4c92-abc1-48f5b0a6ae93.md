# Add cspell for spell checking

## Overview
Integrate [cspell](https://cspell.org/) to catch spelling errors in source code, comments, and documentation. This keeps the codebase professional and consistent.

## Tasks
- Install `cspell` as a dev dependency
- Create `cspell.json` configuration file at the project root
- Configure dictionaries (e.g. `typescript`, `node`, `software-terms`)
- Add custom words list for project-specific terms (e.g. `centy`, `oclif`, `persona`)
- Add a `lint:spell` script in `package.json`: `cspell "**/*.{ts,md,json}"`
- Wire `lint:spell` into the pre-commit hook (after Husky is set up)

## Acceptance Criteria
- Running `npm run lint:spell` reports spelling errors
- False positives for known project terms are suppressed via the custom words list
- The cspell check runs automatically on pre-commit
