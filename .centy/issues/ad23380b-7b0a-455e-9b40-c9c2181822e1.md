# Add commitlint

Set up commitlint to enforce conventional commit message format.

## Tasks
- Install `@commitlint/cli` and `@commitlint/config-conventional`
- Add `commitlint.config.js` (or `.commitlintrc`) with `extends: ['@commitlint/config-conventional']`
- Hook it into `commit-msg` via husky (add or update the hook)

## Acceptance Criteria
- `git commit` with a non-conventional message is rejected
- `git commit` with a valid conventional message (e.g. `feat: add x`) succeeds
