# Configure package for npm publishing

## Overview
Prepare the package for public release on npm as `@centy/persona`.

## Tasks
- Verify `package.json` fields required by npm:
  - `name`, `version`, `description`, `author`, `license`
  - `repository`, `homepage`, `bugs` pointing to the GitHub repo
  - `keywords`: `["centy", "persona", "cli", "ux", "user-story"]`
- Ensure `files` array includes only `dist/` and `README.md`
- Add `README.md` with installation and usage instructions:
  - `npx @centy/persona init`
  - `npm install -g @centy/persona`
- Add a `prepublishOnly` script that runs the build
- Verify the published package size with `npm pack --dry-run`
- Set up a GitHub Actions workflow for automated publishing on tag push (`v*`)

## Acceptance Criteria
- `npm pack --dry-run` shows only expected files
- `npm publish --dry-run` succeeds without errors
- README covers prerequisites (centy daemon running), installation, and the `init` command
