# Add eslint-config-agent for ESLint configuration

## Overview
Set up ESLint using `eslint-config-agent` (or the appropriate shared config for the Centy agent/CLI ecosystem) to enforce consistent code style and catch common errors across the project.

## Tasks
- Install `eslint` and `eslint-config-agent` as dev dependencies
- Create `eslint.config.js` (or `.eslintrc.json`) extending `eslint-config-agent`
- Add TypeScript ESLint support (`@typescript-eslint/parser`, `@typescript-eslint/eslint-plugin`) if not bundled in the config
- Add a `lint` script in `package.json`: `eslint src/`
- Add a `lint:fix` script: `eslint src/ --fix`
- Wire `lint` into the pre-commit hook (after Husky is set up)

## Acceptance Criteria
- Running `npm run lint` reports lint violations
- Running `npm run lint:fix` auto-fixes fixable issues
- No baseline lint errors exist on the main branch
- ESLint runs automatically on pre-commit
