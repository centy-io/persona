# Create centy daemon HTTP client module

## Overview
Build a lightweight typed HTTP client to communicate with the centy daemon. All commands that need to interact with centy (creating item types, querying state, etc.) will use this module.

## Tasks
- Discover the daemon's base URL / socket from the centy daemon info endpoint or environment variable (e.g. `CENTY_DAEMON_URL`, default `http://localhost:<port>`)
- Implement a `CentyClient` class or module with:
  - `get(path, params?)`
  - `post(path, body)`
  - `put(path, body)`
  - Proper error handling — surface meaningful messages when the daemon is not running
- Export the client from `src/client.ts`
- Add `undici` or native `fetch` (Node ≥ 18) as the HTTP transport — no heavy dependencies

## Acceptance Criteria
- Client can reach a running centy daemon and return typed responses
- Meaningful error is thrown/shown when the daemon is unreachable
- No circular dependencies with the commands layer
