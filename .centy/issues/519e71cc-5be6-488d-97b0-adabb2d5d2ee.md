# Set up CLI framework with oclif

## Overview
Scaffold the CLI using [oclif](https://oclif.io/) — the same framework used by centy itself — to ensure consistency and first-class support for commands, flags, and help text.

## Tasks
- Add `@oclif/core` dependency
- Create `src/index.ts` as the CLI entry point (bin)
- Configure `oclif` block in `package.json`:
  - `commands` directory pointing to `src/commands`
  - `bin` name (e.g. `persona` or `centy-persona`)
- Create the `src/commands/` directory structure
- Add a root-level help command / hook
- Wire the bin script so running `persona` from the terminal works after install
- Create `src/daemon-client.ts` — a shared gRPC client module that connects to the centy daemon at `127.0.0.1:50051` using `@grpc/grpc-js` and `@grpc/proto-loader`. All commands that interact with the daemon import from this module.

## Acceptance Criteria
- Running `persona --help` shows the CLI name, version, and available commands
- The commands directory is correctly resolved by oclif
- No TypeScript errors
- `src/daemon-client.ts` exports a usable gRPC client pointed at the centy daemon
